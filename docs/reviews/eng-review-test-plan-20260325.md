## Affected Modules
- `engine/src/hooks/handler.ts` — stdin reader + event router (entry point)
- `engine/src/adapters/claude-code.ts` — Claude Code hook payload → AgentEvent
- `engine/src/events/store.ts` — SQLite append-only event log
- `engine/src/cost/tracker.ts` — session cost aggregation
- `engine/src/escalation/engine.ts` — rule evaluation + alerting

## Key Interactions to Verify
- Claude Code hook fires → handler reads stdin → adapter parses → event stored → cost updated → rules evaluated
- Malformed hook payload → graceful degradation (raw log, no crash)
- Cost threshold exceeded → escalation alert fires
- Missing escalation config → system runs without escalation (no crash)

## Edge Cases
- Empty stdin (hook called with no data)
- Unknown hook event type (future Claude Code updates)
- Disk full during event write (buffer in memory)
- Unknown model ID in cost calculation (fallback pricing)
- Invalid YAML in escalation rules (skip bad rule, log warning)
- Concurrent writes to SQLite (WAL mode handles this — verify)

## Critical Paths
- Full pipeline: hook → adapter → store → cost → escalation (must not throw)
- Adapter graceful degradation: payload format change → null + raw log (never crash the hook)
